# 指标评估阈值（参考）

目的：给出一组启发式的阈值范围，便于判断 `outputs/results` 中各项指标的优劣。注意：阈值受任务、`k` 值、样本数与基线难度影响，请结合具体场景和随机/简单基线一起解读。

总体说明：所有指标均在 0 到 1 之间，值越大表示表现越好。

1) hit@k / recall@k / precision@k（通用分级）

- 优秀 (Excellent): >= 0.75
- 良好 (Good): 0.50 — 0.75
- 中等 (Fair): 0.30 — 0.50
- 较差 (Poor): 0.10 — 0.30
- 不可接受 (Unacceptable): < 0.10

说明与建议：
- 这些范围是通用参考。对于较小的 `k`（如 k=1 或 k=3），期望值通常更低；例如 `hit@1` 的优秀门槛更接近 0.5 而非 0.75。
- 当比较不同 `k` 或 `horizon` 时，优先关注相同 `k` 下的变化趋势。

2) MRR（Mean Reciprocal Rank）

- 优秀: >= 0.70
- 良好: 0.40 — 0.70
- 中等: 0.20 — 0.40
- 较差: 0.05 — 0.20
- 不可接受: < 0.05

说明：MRR 强调首个正确项的排名，若应用场景更看重前几名排序，应把 MRR 作为主要参考指标。

3) `num_samples`（样本量可靠性指示）

- 稳定: >= 200
- 可接受: 100 — 200
- 较低: 50 — 100（需谨慎）
- 很低: < 50（结果方差大，不宜单独依赖）

4) 其它建议（实践要点）

- 总是与随机基线或最简单基线（例如按频率预测）对比，若提升微小即便绝对值看起来“良好”也可能不可用。 
- 报告指标时同时给出样本数与方差／置信区间（若可能）。
- 若业务有明确的最低可接受值（SLA），以业务阈值为准；将此处阈值作为参考帮助快速筛选实验优劣。

示例：对于 `app_markov_results.csv` 中的 `k=5` 行，如果 `hit@5≈0.33`，那么按上表可判为“中等/良好之间”（接近良好上限）；若 `num_samples` 只有 52，则结果不够稳定，需谨慎解读。
