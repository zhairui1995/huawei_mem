# 第一阶段最小实验流程文件说明

本文档说明第一阶段用户行为预测实验中新增和修改的文件。第一阶段只实现模拟数据生成、样本构造、时间切分、Markov baseline 和评估指标，不包含 LSTM。

## 一键运行入口

### `scripts/run_first_stage.sh`

第一阶段完整流水线入口。

执行顺序：

1. 生成模拟应用日志和应用内操作日志。
2. 构造应用间预测样本。
3. 构造应用内操作预测样本。
4. 按时间顺序切分应用间数据集。
5. 按时间顺序切分应用内数据集。
6. 训练并评估应用间 Markov baseline。
7. 训练并评估应用内 Markov baseline。

推荐运行命令：

```bash
bash scripts/pipeline/run_first_stage.sh
```

运行完成后会生成 `data/raw/`、`data/processed/` 和 `outputs/results/` 下的第一阶段产物。

## 模拟数据生成

### `scripts/data/make_synthetic_data.py`

生成第一版实验使用的模拟用户行为日志。

读取：

- `data/vocab/app_vocab.json`
- `data/vocab/op_vocab.json`
- `data/vocab/user_group_vocab.json`

输出：

- `data/raw/app_events.csv`
- `data/raw/op_events.csv`

主要逻辑：

- 默认生成 10 个用户、每个用户约 2 小时行为。
- 应用切换间隔随机为 1 到 5 分钟。
- 应用内操作间隔随机为 5 到 60 秒。
- 每个应用会话内生成 3 到 10 个操作。
- `opened_apps` 使用应用生命周期状态生成：每个用户初始没有打开应用，每次最多新打开一个应用，未关闭的应用会持续保留，随机关闭时也只关闭一个非前台应用。
- 使用固定随机种子，保证可复现。
- 根据用户类型给部分应用加权，例如办公人群偏向 WPS、飞书、腾讯 QQ 等。
- 如果 `user_group_vocab.json` 不存在，会自动创建默认版本。

示例：

```bash
python scripts/data/make_synthetic_data.py \
  --num-users 10 \
  --hours 2 \
  --seed 42 \
  --output-app data/raw/app_events.csv \
  --output-op data/raw/op_events.csv
```

## 数据样本构造

### `src/data/build_app_dataset.py`

把 `app_events.csv` 转成应用间预测样本。

读取：

- `data/raw/app_events.csv`
- `data/vocab/app_vocab.json`
- `data/vocab/user_group_vocab.json`

输出：

- `data/processed/app_samples.pkl`

每个样本包含：

- `user_id`
- `timestamp`
- `time_feature`
- `user_group`
- `history_apps`
- `opened_apps`
- `label_3min`
- `label_5min`
- `label_10min`

主要逻辑：

- 按 `user_id` 分组，并按 `timestamp` 排序。
- 默认用过去 5 个 `foreground_app` 作为 `history_apps`。
- 把 `opened_apps` 转成应用 multi-hot 向量。
- 把未来 3、5、10 分钟内出现过的前台应用转成 multi-hot 标签。
- 如果未来窗口没有任何应用，则跳过该样本。
- 如果应用不在 `app_vocab.json` 中，则跳过该事件。
- 如果用户组不在词表中，则回退为 `通用用户`。

### `src/data/build_op_dataset.py`

把 `op_events.csv` 转成应用内下一步操作预测样本。

读取：

- `data/raw/op_events.csv`
- `data/vocab/op_vocab.json`
- `data/vocab/user_group_vocab.json`

输出：

- `data/processed/op_samples.pkl`

每个样本包含：

- `user_id`
- `timestamp`
- `app`
- `time_feature`
- `user_group`
- `history_ops`
- `label_next_op`

主要逻辑：

- 按 `user_id + app` 分组，并按 `timestamp` 排序。
- 默认用过去 4 个操作 ID 作为 `history_ops`。
- 使用 `op_vocab.json[app]` 做操作映射。
- 未知操作映射为 `<UNK>`。
- `<PAD>` 不作为真实操作，也不会作为标签参与训练。
- 如果应用不在 `op_vocab.json` 中，则跳过该事件。
- 如果用户组不在词表中，则回退为 `通用用户`。

### `src/data/split_dataset.py`

按时间顺序切分应用间或应用内样本。

读取：

- `data/processed/app_samples.pkl` 或 `data/processed/op_samples.pkl`

输出：

- `data/processed/train_app.pkl`
- `data/processed/val_app.pkl`
- `data/processed/test_app.pkl`
- `data/processed/train_op.pkl`
- `data/processed/val_op.pkl`
- `data/processed/test_op.pkl`

主要逻辑：

- 按样本中的 `timestamp` 排序。
- 默认比例为 train 70%、val 15%、test 15%。
- 不做随机切分，避免未来数据泄漏到训练集。

## Markov 模型

### `src/models/app_markov.py`

应用间 Markov baseline 模型。

模型能力：

- 支持一阶到四阶 Markov。
- 输入 `history_apps`。
- 使用最近 `order` 个应用作为上下文。
- 按预测窗口分别维护转移表，即 `horizon -> context -> app`。
- 训练时只统计最后 `order` 个历史应用到未来窗口标签的关系，不统计 `history_apps` 内部转移。
- 全局退避频率只统计未来标签出现频率，不混入历史应用出现频率。
- 预测时输出指定 horizon 下的应用 Top-K。
- 如果高阶上下文未出现过，则逐级退避到低阶上下文；仍未出现时回退到该 horizon 的全局未来标签频率。

### `src/models/op_markov.py`

应用内操作 Markov baseline 模型。

模型能力：

- 每个应用单独维护操作转移统计。
- 支持一阶到四阶 Markov。
- 输入 `app` 和 `history_ops`。
- 使用最近 `order` 个操作作为上下文。
- 如果高阶上下文未出现过，则逐级退避到低阶上下文；仍未出现时回退到该应用下的全局操作频率。
- 预测时输出下一步操作 Top-K。
- 如果上下文未出现过，则回退到该应用下的全局操作频率。
- 不把 `<PAD>` 作为预测结果。
- 尽量不把 `<UNK>` 排在前面，除非没有其他候选。

## 训练与评估脚本

### `src/train/train_app_markov.py`

训练并评估应用间 Markov baseline。

读取：

- `data/processed/train_app.pkl`
- `data/processed/test_app.pkl`

输出：

- `outputs/results/app_markov_results.csv`

输出 CSV 字段：

- `model`
- `order`
- `horizon`
- `k`
- `hit_at_k`
- `recall_at_k`
- `precision_at_k`
- `mrr`
- `num_samples`

评估内容：

- 分别评估 3、5、10 分钟未来窗口。
- 默认评估 Top-1、Top-3、Top-5。
- 指标包括 Hit@K、Recall@K、Precision@K 和 MRR。

### `src/train/train_op_markov.py`

训练并评估应用内操作 Markov baseline。

读取：

- `data/processed/train_op.pkl`
- `data/processed/test_op.pkl`
- `data/vocab/op_vocab.json`

输出：

- `outputs/results/op_markov_results.csv`

输出 CSV 字段：

- `app`
- `model`
- `order`
- `k`
- `topk_accuracy`
- `mrr`
- `num_samples`

评估内容：

- 对每个应用分别计算 Top-K Accuracy 和 MRR。
- 额外输出一行 `app=ALL` 的整体平均结果。
- 默认评估 Top-1、Top-3、Top-5。

## 评估指标

### `src/eval/metrics.py`

通用评估指标实现。

函数：

- `hit_at_k(pred_topk, labels)`
- `recall_at_k(pred_topk, labels)`
- `precision_at_k(pred_topk, labels)`
- `mrr(pred_ranked_list, labels)`
- `topk_accuracy(pred_topk, label)`

兼容情况：

- `labels` 可以是 multi-hot 向量。
- `labels` 也可以是 ID 集合或列表。

该文件带有简单自测，可以运行：

```bash
python src/eval/metrics.py
```
